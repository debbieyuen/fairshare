# Android Capacitor Workflow

This repo uses Capacitor for the native Android app. The web source of truth
lives at the repo root; `www/` is generated by `npm run build`.

## Windows 11 Setup

1. Install **Node.js 18+** (LTS):

   ```powershell
   winget install OpenJS.NodeJS.LTS
   ```

   Open a new terminal after install so `node` and `npm` are on PATH.

2. Install **Android Studio** from [developer.android.com/studio](https://developer.android.com/studio)
   (or `winget install Google.AndroidStudio`).

3. Install Android SDK Platform 34 and tools (one-time, headless):

   ```powershell
   .\scripts\install-android-sdk.ps1
   ```

   Or use Android Studio **SDK Manager** manually.

4. In Android Studio **SDK Manager**, confirm installed:

   - Android SDK Platform **34** (Android 14)
   - Android SDK Build-Tools (34.x)
   - Android SDK Platform-Tools

5. Optional but recommended: set `ANDROID_HOME` so CLI tools find the SDK:

   ```powershell
   [Environment]::SetEnvironmentVariable('ANDROID_HOME', "$env:LOCALAPPDATA\Android\Sdk", 'User')
   ```

6. Create an emulator in **Device Manager**, or enable **Developer options** and
   **USB debugging** on a physical Android device.

7. Verify toolchain from repo root:

   ```powershell
   .\scripts\check-android-toolchain.ps1
   ```

8. Install dependencies:

   ```powershell
   npm install
   ```

The build script uses a cross-platform Node copy (`scripts/sync-www.js`) — no
`rsync` or WSL required on Windows.

Before running `npm`, `adb`, or `gradlew.bat` in PowerShell, load paths for
this session (Node is installed but may not appear until a new terminal, or
after winget install):

```powershell
. .\scripts\android-env.ps1
```

This sets `JAVA_HOME`, `ANDROID_HOME`, and adds Node + `platform-tools` (adb) to
PATH. Alternatively, open a **new** terminal after installing Node/Android SDK.

### Command-line Gradle (optional)

Android Studio uses its bundled JDK automatically. For `gradlew.bat` outside
Studio, point `JAVA_HOME` at Studio's JBR:

```powershell
$env:JAVA_HOME = "$env:ProgramFiles\Android\Android Studio\jbr"
```

SDK default location: `%LOCALAPPDATA%\Android\Sdk`

## Mac Setup

1. Install Node.js 18 or newer.
2. Install dependencies from the repo root:

   ```bash
   npm install
   ```

3. Install Android Studio.
4. In Android Studio, install Android SDK Platform Tools, Android SDK Build
   Tools, and an Android SDK platform. This project currently targets SDK 34.
5. Use Android Studio's bundled JDK. If command-line tools cannot find Java:

   ```bash
   export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
   ```

6. Create an emulator in Android Studio Device Manager, or enable Developer
   Options and USB debugging on a physical Android device.

## Development Build

Run this after changing HTML, CSS, JS, Capacitor config, or native plugins:

```bash
npm run cap:sync:android
npx cap open android
```

Build and run from Android Studio, or use:

```bash
npx cap run android
```

On Windows, `npx cap open android` opens the `android/` project in Android
Studio. Click **Run** on an emulator or USB device.

After any web asset change, always re-sync before rebuilding. If assets look
stale in Studio: **Build → Clean Project**, then Run again.

## Firebase Cloud Messaging

Android native push uses Firebase Cloud Messaging.

1. Create an Android app in [Firebase Console](https://console.firebase.google.com/)
   with package name `social.fairshare.union`.
2. Download `google-services.json`.
3. Put it at `android/app/google-services.json` (see
   `android/app/google-services.json.example` for shape; do not commit the real
   file).
4. Configure the Supabase native push edge function with FCM credentials (see
   **Backend FCM secrets** below).

The existing native push edge function routes `platform: 'ios'` device tokens
to APNs and `platform: 'android'` tokens to FCM.

Until `google-services.json` is present, the web app does **not** call
`PushNotifications.register()` on Android (so login and the rest of the app
keep working). After you add the file, do a clean rebuild so
`BuildConfig.HAS_GOOGLE_SERVICES` becomes true.

### Verify token registration on device

1. Run `npm run cap:sync:android`, rebuild in Android Studio.
2. Sign in (push auto-subscribes when `profiles.push_notifications !== false`).
3. Grant the **Notifications** permission when prompted.
4. In **Logcat**, expect (use tag **Capacitor/Console** or search `Msg:` — a
   plain `push` filter often shows nothing because `console.log` goes through
   Capacitor, not a `push` tag):

   - `[push] Permission result: ...`
   - `[push] register() called successfully`
   - `[push] Native device token: android <fcm-token>`
   - `[push] Device token saved successfully`

5. In Supabase, confirm a row in `device_push_tokens` with `platform = 'android'`.

### Backend FCM secrets

For the edge function to **send** pushes (not just register tokens on device):

1. Firebase Console → **Project settings → Service accounts** → **Generate new
   private key** (JSON file).
2. From repo root, print the Supabase CLI commands (does not upload secrets
   itself):

   ```powershell
   node scripts/setup-fcm-secrets.js path\to\firebase-service-account.json
   ```

3. Run the printed `supabase secrets set ...` commands, then:

   ```bash
   supabase functions deploy send-push-apns
   ```

4. Ensure Supabase Vault has `apns_edge_fn_url` pointing to the deployed
   `send-push-apns` function URL (same function handles iOS and Android).

5. Ensure [sql/apns-push-schema.sql](../sql/apns-push-schema.sql) has been
   applied (creates `device_push_tokens` and wires native delivery).

### End-to-end push test

Trigger any push your account should receive (e.g. a group chat message from
another user). Tap the notification; deep links route via `handleNotificationNavigation`
in [js/push.js](../js/push.js).

## Troubleshooting

### `Default FirebaseApp is not initialized` (CapacitorPlugins crash)

That means native push ran without Firebase config. Add
`android/app/google-services.json` from the Firebase console (see **Firebase
Cloud Messaging** above), then rebuild. If you hit this during development
before Firebase is set up, pull the latest app code: native push registration is
skipped on Android until `google-services.json` exists.

### App works but no push registration logs

`window.__UNION_ANDROID_FCM__` is still false — add `google-services.json` and
do a clean rebuild.

### `npm run build` fails

Ensure Node.js 18+ is installed and on PATH. The build no longer requires rsync.

### Gradle / JDK errors

Open the project in Android Studio and use its bundled JDK. For CLI builds, set
`JAVA_HOME` to Android Studio's `jbr` folder (see Windows section above).

### Token saved but no delivery

Check FCM secrets on the `send-push-apns` edge function and the
`apns_edge_fn_url` vault secret in Supabase.

### Chromium GPU / keyboard (`EGL_BAD_PARAMETER`, `SharedImageManager`)

Those lines usually come from the system WebView’s compositor when the soft
keyboard opens or the viewport resizes. This project sets
`android:windowSoftInputMode="adjustPan"` on `MainActivity` and sizes the group
chat panel using `visualViewport` so layout does not fight `window.innerHeight`.
If a specific device still glitches, try changing that attribute to
`adjustResize` in `android/app/src/main/AndroidManifest.xml` and rebuild.

## Release Build (Google Play)

Use Play App Signing with a local upload key. Keep keystores and passwords out
of git.

### Prerequisites

1. [Google Play Console](https://play.google.com/console) developer account
   ($25 one-time).
2. Upload keystore created locally (Android Studio can generate one during
   **Generate Signed Bundle**).

### Build steps

1. Always sync web assets first:

   ```bash
   npm run cap:sync:android
   ```

2. Android Studio → **Build → Generate Signed Bundle / APK** → **Android App
   Bundle (.aab)** → use your upload keystore.

3. Upload the `.aab` to Play Console (Internal testing track is enough for
   production FCM validation before public release).

### Play Console declarations

Review and declare permissions already in
[AndroidManifest.xml](../android/app/src/main/AndroidManifest.xml):

- `INTERNET`
- `CAMERA`
- `ACCESS_COARSE_LOCATION` / `ACCESS_FINE_LOCATION`
- `POST_NOTIFICATIONS` (Android 13+)

Complete **Data safety** and content rating questionnaires before submitting.
